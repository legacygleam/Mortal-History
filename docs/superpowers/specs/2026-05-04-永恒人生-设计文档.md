# 永恒人生（EternalLife）—— 设计文档

## 概述

一个去中心化、公益性质的个人历史永久保存平台。让每个人——不只是伟人和企业家——的生命故事、经验教训、人生智慧被永久铭记，为后人提供宝贵的人生指南。

核心理念：**底层数据不可篡改，审核只在发现层；不依赖任何单一实体。**

---

## 一、核心数据模型

### 1.1 文件集合结构

每个人的生命档案是一个文件集合，以 Markdown 纯文本格式存储：

```
{用户DID}/
├── profile.md              # 基本档案
├── languages.json          # 多语言声明
├── biography/              # 传记（按时期分章）
│   ├── 00-童年.md
│   ├── 01-青年.md
│   ├── 02-中年.md
│   └── 03-晚年.md
├── experiences/            # 经验教训条目
│   ├── 2026-0504-职场沟通.md
│   ├── 2026-0510-投资教训.md
│   └── ...
├── wisdom/                 # 人生建议（精选短条目）
│   ├── 给年轻人的话.md
│   └── ...
├── timeline.md             # 大事记时间线
└── assets/                 # 图片附件（压缩后）
    └── *.webp
```

### 1.2 加密确权

每个文件头部包含 YAML frontmatter：

```yaml
---
author: <用户 DID>
signed: <GPG/钱包签名>
timestamp: 2026-05-04T10:00:00Z
language: zh
---
```

签名确保：任何人可验证内容确为用户所写，无法伪造。

### 1.3 多语言支持

```
{用户DID}/
├── zh/              # 中文版（主语言）
│   └── biography/...
├── en/              # 英文版（翻译）
│   └── biography/...
└── languages.json   # 语言声明
```

主语言为作者母语。翻译可由本人、后人或社区协作完成。每个翻译版有独立签名。

### 1.4 图片处理

- 前端自动压缩至 ≤500KB，转换为 WebP 格式
- 图片与文本分开上传到 Arweave，文本中引用 txID
- 原图不保留（控制存储成本）

## 二、存储架构

### 2.1 技术选型

| 层 | 技术 | 角色 | 费用 |
|---|------|------|------|
| 主存储 | Arweave | 永久存档，不可篡改 | 一次性付费（~0.5元/篇） |
| 辅助分发 | IPFS | 快速读取，内容寻址 | 免费 |
| 用户导出 | Git | 可导出推送到 GitHub/Gitee | 免费 |

### 2.2 数据写入流程

```
用户编写（Web 前端）
    │
    ▼
用户钱包签名（确权）
    │
    ▼
        ┌──────── 多重分发 ────────┐
        ↓                          ↓
    Arweave（永久主存）          IPFS（辅助分发）
        ↓                          ↓
    返回 txID                  返回 CID
    │
    ▼
索引服务抓取 → 审核 → 收录到发现层
```

### 2.3 数据读取流程

```
用户访问某个档案
    │
    ▼
前端通过 Arweave 网关读取：https://arweave.net/{txID}
    │
    ▼
配合 IPFS 网关做 CDN 加速
    │
    ▼
渲染为可读页面
```

### 2.4 导出能力

用户随时可：
- 一键导出 ZIP（纯 Markdown + 图片），完全独立于平台
- 推送到 Git 仓库（额外备份）
- 打印为 PDF（纸质存档）

## 三、身份与认证系统

### 3.1 去中心化身份

- 通过 **ArConnect 钱包**（浏览器插件）管理密钥
- 公钥 = DID（去中心化 ID）
- 私钥签名每一篇内容
- 助记词（12 词）需用户安全备份

### 3.2 用户流程

```
首次使用：
  安装 ArConnect → 创建身份 → 备份助记词

日常使用：
  写完内容 → 点击发布 → 钱包弹窗签名 → 自动上传

阅读他人档案：
  无需任何身份，直接通过网关访问
```

### 3.3 社交恢复机制

用户可指定 3-5 位"守护人"（家人/朋友）。若丢失密钥，超半数守护人可协助恢复身份控制权。历史内容仍由旧密钥签名不可变，仅控制权转移。

### 3.4 身份信息

```
profile.md 中：
  - 真实姓名: 可选
  - 笔名/昵称: 必填
  - 生卒年月: 必填
  - 国籍/地区: 可选
  - 公开范围: 公开/指定人可见（未来功能）
```

## 四、前端应用

### 4.1 技术架构

- 纯静态 Web 应用（HTML/CSS/JS），无后端服务器
- 部署于 Arweave Permaweb
- 所有数据运行时从 Arweave 网关获取
- 即使开发者消失，应用仍永久运行

### 4.2 页面结构

#### 首页（发现/浏览）
- 搜索框（按姓名、国籍、年代、标签）
- 最近收录
- 随机推荐（让每个生命都被看见）
- 分类浏览（年代 / 地区 / 职业）

#### 个人档案页
- 基本信息（头像、姓名、生卒年、简介）
- 传记（目录化展示，可翻页阅读）
- 经验教训（分类展示，可筛选）
- 人生建议（精选摘要卡片）
- 时间线（可视化）

#### 写作台（编辑器）
- 所见即所得 Markdown 编辑器
- 图片上传（自动压缩至 WebP, ≤500KB）
- 语言版本管理
- 预览功能
- 一键发布（钱包签名确认）

### 4.3 发现层与审核

```
用户发布 → Arweave 永久存储（不可删，不审核）
         ↓
         索引服务抓取新内容
         ↓
         审核规则（AI 自动 + 人工举报）
         ↓
         合规 → 收录到公开索引
         违规 → 不收录，但直接链接仍可访问
```

- 数据层永久不可篡改，不设审核
- 审核仅在发现/搜索/推荐层
- 全球可运行多个索引服务，用户自由选择信任哪个索引

## 六、原则

1. **数据不锁定**——任何时刻用户可导出全部数据，格式标准
2. **审核不删改**——底层数据永远不可篡改
3. **渐进去中心**——从实用的半中心化索引起步，逐步社区化
4. **极简成本**——一篇传记的永久存储成本控制在几元钱
5. **面向百年**——纯文本格式确保几百年后仍可阅读
